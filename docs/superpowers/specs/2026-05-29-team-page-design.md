# 대웅찬넷 팀 소개 웹페이지 — 설계 문서

작성일: 2026-05-29

## 배경 (Context)

부산대학교 학생 창업팀 **(주)대웅찬넷**(정선웅·한대희·정의찬, 4년지기 친구)이 6개월간 소프트웨어 개발 프로젝트를 진행한다.
그 시작점으로, 팀과 아이디어를 소개하는 웹페이지를 만든다. 멘토·외부에 한 장으로 팀과 아이디어를 전달하는 것이 목표다.

원본 정보는 `team.md`와 팀 소개 PPT(`대웅찬넷_팀소개.pptx`)에 있으며, 본 페이지는 PPT의 핵심(팀 정체성·팀원·강점·아이디어 ①②)을 웹으로 옮긴 "핵심 확장" 버전이다.

## 목표 (Goals)

- 팀 정체성((주)대웅찬넷, 부산대 학생 창업팀)과 슬로건을 명확히 전달
- 팀원 3명을 깔끔한 카드로 소개
- 팀 강점 6가지를 한눈에 보여줌
- 두 가지 아이디어(① AI 내장 캘린더, ② 화면 분할 프리셋)를 소개 — ①을 메인으로
- 누구나(개발 경험 적은 팀원 포함) 쉽게 내용을 수정할 수 있을 것
- GitHub Pages 등에 바로 배포 가능할 것

## 비목표 (Non-Goals)

- 실제 AI 캘린더 제품/기능 구현 (이 페이지는 소개용)
- 백엔드, 데이터베이스, 인증
- CMS, 다국어 (한국어 단일)
- 빌드 도구/프레임워크 도입

- 외부 의존성 없음 (CDN/프레임워크 미사용). 폰트는 시스템 폰트 스택 사용
- 정적 파일이므로 GitHub Pages에 그대로 배포 가능

### 4. 아이디어 (`#ideas`)
두 가지 아이디어를 소개. ①을 메인(풀 구성), ②를 보조 카드로.

**아이디어 ① · AI 내장 캘린더 (메인)** — "문제 → 다리 문구 → 해결" 구성

- **문제 (THE PROBLEM) — 기존 캘린더, 너무 불편하다**
  - 폰·데탑·맥북·애플워치·아이패드 — 기기 간 동기화가 제각각
  - 일정 하나 등록하는 데 날짜·시간·태그·위치·반복·알림… 입력이 너무 많음
  - 기간에 걸친 일정이나 중간에 끊기는 일정은 여러 번 등록해야 함
- **다리 문구 (왜 직접 만드나)**
  - "구글 캘린더 + AI 에이전트 조합이면 가능은 하지만, 일반 사용자에게는 그 세팅 자체가 진입장벽입니다." → 그래서 AI를 처음부터 내장한 캘린더를 만든다.
- **해결 (OUR SOLUTION) — 자연어 딸깍, 스크린샷 딸깍**
  - 설명: "내일 3시에 회의" 한 줄, 혹은 포스터 스크린샷 한 장이면 AI 에이전트가 일정을 알아서 등록·정리
  - 특징 태그: 자연어 입력 / 스크린샷 입력 / 캘린더+할 일 통합 / 멀티 디바이스 동기화 / 멀티데이·비연속 일정 지원

**아이디어 ② · 화면 분할 프리셋 (보조)**
- 한 줄: "단축키 한 번에 '작업 모드'가 펼쳐지는 윈도우 매니저"
- 문제: 작업 시작마다 같은 창 배치 반복 / 패턴인데 매번 수작업 / 기존 매니저는 '배치'만, '앱 실행+배치'는 약함
- 해결: 프리셋 + 단축키 = 한 방에 세팅 (예: CP 프리셋 = Codeforces + VS Code, 메이플 프리셋 = 메인 게임 + 서브 모니터 유튜브)

## 파일 구조

```
ASMteamPage/
  index.html      # 전체 페이지 (HTML + 내장 <style>, 필요 시 작은 <script>)
  team.md         # 원본 정보 (소스 of truth)
```

- CSS는 `index.html` 내 `<style>`에 둔다 (단일 파일 유지). 길어지면 `styles.css`로 분리 가능.
- JS는 거의 불필요(앵커 스크롤은 CSS `scroll-behavior`로 처리). 모바일 네비 토글 정도만 필요 시 소량 추가.
